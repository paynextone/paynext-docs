Integrate PayNext
into your product

PayNext is a subscription billing platform for SaaS products. This guide shows you how to connect your application — whether built with Node.js, Next.js, PHP, or Laravel — to accept payments and manage subscriptions in minutes.

▲ Next.js

How It Works

PayNext sits between your app and the payment gateway. You handle the user experience — PayNext handles billing, subscriptions, and payments.

Your App
Evalios / Matix
→
PayNext API
HMAC Signed
→
Checkout Page
Hosted by PayNext
→
Tap Payments
3DS Auth
→
✓ Subscribed
Webhook fired
💡
You never handle card data. PayNext and Tap Payments handle all card processing. Your server only makes API calls and receives webhooks.

Quick Start

Get a working subscription flow in 5 steps. Follow them in order.

1
Get your API credentials
Log in to the PayNext Admin Portal → open your App → copy your API Key and API Secret. Store them as environment variables — never hardcode them.
.env
PAYNEXT_API_KEY=pk_live_xxxxxxxxxxxxxxxxxxxx
PAYNEXT_API_SECRET=sk_live_xxxxxxxxxxxxxxxxxxxx
PAYNEXT_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxxxxxxxxx
2
Install the SDK
Terminal
npm install paynext-sdk
Terminal
# Via Composer
composer require paynext/sdk

# Or copy src/PayNextClient.php into your project's lib/ folder
3
Initialize the client
JavaScript
const { PayNextClient } = require('paynext-sdk');

const paynext = new PayNextClient({
  apiKey:    process.env.PAYNEXT_API_KEY,
  apiSecret: process.env.PAYNEXT_API_SECRET,
  baseUrl:   'https://paynext.one',
});
lib/paynext.js
// Server-side only — never import in browser components
import { PayNextClient } from 'paynext-sdk';

export const paynext = new PayNextClient({
  apiKey:    process.env.PAYNEXT_API_KEY,
  apiSecret: process.env.PAYNEXT_API_SECRET,
});
PHP
require_once 'vendor/autoload.php';
use PayNext\PayNextClient;

$paynext = new PayNextClient(
    apiKey:    $_ENV['PAYNEXT_API_KEY'],
    apiSecret: $_ENV['PAYNEXT_API_SECRET']
);
4
Create a checkout session & redirect
When a user clicks "Subscribe", call this on your server and redirect them to the returned URL.
JavaScript
const session = await paynext.checkout.createSession({
  customer_id:   'CUS-XXXXXX',
  plan_id:       'PLAN-XXXXXX',
  billing_cycle: 'monthly',  // 'monthly' | 'yearly'
  success_url:   'https://yourapp.com/success',
  cancel_url:    'https://yourapp.com/pricing',
});

res.redirect(session.data.checkout_url);
5
Listen for webhooks to activate features
After payment succeeds, PayNext fires a subscription.activated event to your endpoint. Use it to unlock premium features.
JavaScript
app.post('/webhooks/paynext', express.raw({ type: '*/*' }), (req, res) => {
  const valid = PayNextClient.verifyWebhook(
    req.body,
    req.headers['x-paynext-signature'],
    process.env.PAYNEXT_WEBHOOK_SECRET
  );
  if (!valid) return res.sendStatus(401);

  const event = JSON.parse(req.body);
  if (event.type === 'subscription.activated') {
    // Unlock features for event.data.customer.external_id
  }
  res.sendStatus(200);
});

Authentication

Every API request must be signed with your API Key and Secret using HMAC-SHA256. The SDK handles this automatically.

⚠️
API Secret is server-side only. Never expose your PAYNEXT_API_SECRET to the browser, mobile app, or public repository. All PayNext API calls must be made from your backend server.

Required Headers on Every Request

HeaderValue
AuthorizationBearer YOUR_API_KEY
X-SignatureHMAC-SHA256 of: METHOD\nPATH\nTIMESTAMP\nBODY
X-TimestampUnix timestamp in seconds — requests older than 5 min are rejected

Customers

A Customer represents an end-user of your product. Always pass your own user ID as external_id — this lets you look up PayNext customers without storing extra IDs in your database.

✅
Best Practice: Get-or-Create pattern. Before checkout, try getByExternalId(userId) first. Only create a new customer if they don't exist yet.
JavaScript
async function getOrCreate(user) {
  try {
    const res = await paynext.customers.getByExternalId(user.id);
    return res.data;
  } catch {
    const res = await paynext.customers.create({
      name:        user.name,
      email:       user.email,
      phone:       user.phone,       // e.g. '+966501234567'
      country:     user.country,     // ISO 2-letter: 'SA', 'AE', 'EG'
      currency:    user.currency,    // ISO 3-letter: 'SAR', 'AED', 'EGP'
      external_id: String(user.id),  // YOUR own user ID
    });
    return res.data;
  }
}
PHP
function getOrCreate($paynext, $user): array {
    try {
        return $paynext->customers->getByExternalId((string) $user['id'])['data'];
    } catch (PayNextException $e) {
        return $paynext->customers->create([
            'name'        => $user['name'],
            'email'       => $user['email'],
            'country'     => $user['country'] ?? 'SA',
            'currency'    => $user['currency'] ?? 'SAR',
            'external_id' => (string) $user['id'],
        ])['data'];
    }
}

Checkout & Payments

PayNext hosts the checkout page. You create a session, redirect the user, and PayNext handles the card form, 3DS authentication, and payment processing.

Session Parameters

ParameterTypeRequiredDescription
customer_idstringRequiredPayNext Customer ID (CUS-XXXXXX)
plan_idstringRequiredPayNext Plan ID
billing_cyclestringRequiredmonthly or yearly
success_urlstringOptionalRedirect here after successful payment
cancel_urlstringOptionalRedirect here if user cancels
discount_codestringOptionalPre-apply a discount code
⏱️
Sessions expire after 1 hour. If the user doesn't complete payment in time, create a new checkout session.

Subscriptions

Check and manage customer subscriptions programmatically.

JavaScript
// Check active subscription
const subs   = await paynext.subscriptions.list({ customer_id: customer.id });
const active = subs.data.find(s => s.status === 'active');

// Cancel at period end
await paynext.subscriptions.cancel(subscriptionId, { cancel_at_period_end: true });

// Pause / Resume
await paynext.subscriptions.pause(subscriptionId);
await paynext.subscriptions.resume(subscriptionId);
PHP
$subs   = $paynext->subscriptions->list(['customer_id' => $customerId]);
$active = null;
foreach ($subs['data'] as $sub) {
    if ($sub['status'] === 'active') { $active = $sub; break; }
}

// Cancel at period end
$paynext->subscriptions->cancel($subscriptionId, ['cancel_at_period_end' => true]);

Webhooks

Webhooks notify your server in real-time when events happen — subscription activated, payment succeeded, etc. This is how you know when to unlock or revoke features.

1Go to PayNext Admin → Webhooks → Add Endpoint and enter your server URL, e.g. https://yourapp.com/webhooks/paynext
2Select the events you want, then copy the generated Webhook Secret
3Add it to your environment as PAYNEXT_WEBHOOK_SECRET
4Create a route and verify the signature on every incoming request
⚠️
Important: Use express.raw() in Node.js or file_get_contents('php://input') in PHP to read the raw body before JSON parsing. Signature verification requires the exact raw bytes.
JavaScript
app.post('/webhooks/paynext',
  express.raw({ type: 'application/json' }),
  (req, res) => {
    const valid = PayNextClient.verifyWebhook(
      req.body,
      req.headers['x-paynext-signature'],
      process.env.PAYNEXT_WEBHOOK_SECRET
    );
    if (!valid) return res.status(401).end();

    const event = JSON.parse(req.body);
    switch (event.type) {
      case 'subscription.activated':
        // Unlock features for event.data.customer.external_id
        break;
      case 'subscription.cancelled':
      case 'subscription.expired':
        // Revoke features
        break;
      case 'payment.succeeded':
        // Send receipt email
        break;
    }
    res.sendStatus(200);
  }
);
app/api/webhooks/paynext/route.js
import { NextResponse } from 'next/server';
import { PayNextClient } from 'paynext-sdk';

export async function POST(request) {
  const rawBody   = await request.text();
  const signature = request.headers.get('x-paynext-signature');
  const valid = PayNextClient.verifyWebhook(
    rawBody, signature, process.env.PAYNEXT_WEBHOOK_SECRET
  );
  if (!valid) return NextResponse.json({ error: 'Invalid' }, { status: 401 });

  const event = JSON.parse(rawBody);
  // Handle event.type ...
  return NextResponse.json({ received: true });
}
PHP
$rawBody   = file_get_contents('php://input');
$signature = $_SERVER['HTTP_X_PAYNEXT_SIGNATURE'] ?? '';

if (!PayNextClient::verifyWebhook($rawBody, $signature, $_ENV['PAYNEXT_WEBHOOK_SECRET'])) {
    http_response_code(401); exit;
}
$event = json_decode($rawBody, true);

switch ($event['type']) {
    case 'subscription.activated':
        // Activate for $event['data']['customer']['external_id']
        break;
}
http_response_code(200);
echo json_encode(['received' => true]);

Event Types

All events PayNext can send to your webhook endpoint.

subscription.activated
Payment captured. Subscription is now active.
subscription.created
A new subscription record was created.
subscription.renewed
Billing cycle renewed successfully.
subscription.cancelled
Cancelled by admin or user.
subscription.expired
Period ended without renewal.
subscription.paused
Subscription was paused.
subscription.resumed
Paused subscription was resumed.
payment.succeeded
A payment was captured successfully.
payment.failed
A payment attempt failed.
payment.refunded
A payment was refunded.
