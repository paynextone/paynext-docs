دمج PayNext
في منتجك

PayNext منصة فوترة اشتراكات مخصصة لمنتجات SaaS. يوضح هذا الدليل كيفية ربط تطبيقك — سواء كان مبنيًا بـ Node.js أو Next.js أو PHP أو Laravel — لقبول المدفوعات وإدارة الاشتراكات في دقائق.

🟢 Node.js
▲ Next.js
🐘 PHP / Laravel
🔒 محمي بـ HMAC-SHA256
🔔 Webhooks

كيف يعمل

يقع PayNext بين تطبيقك وبوابة الدفع. أنت تتولى تجربة المستخدم — وPayNext يتولى الفوترة والاشتراكات والمدفوعات.

البداية السريعة

احصل على تدفق اشتراك يعمل في 5 خطوات. اتبع الترتيب بدقة.

1
احصل على بيانات اعتماد API
سجّل الدخول إلى بوابة PayNext الإدارية ← افتح تطبيقك ← انسخ API Key وAPI Secret. احفظهما كمتغيرات بيئة — لا تضعهما مباشرةً في الكود أبدًا.
.env
PAYNEXT_API_KEY=pk_live_xxxxxxxxxxxxxxxxxxxx
PAYNEXT_API_SECRET=sk_live_xxxxxxxxxxxxxxxxxxxx
PAYNEXT_WEBHOOK_SECRET=whsec_xxxxxxxxxxxxxxxxxxxx
2
ثبّت الـ SDK
Terminal
npm install paynext-sdk
Terminal
# عبر Composer
composer require paynext/sdk

# أو انسخ src/PayNextClient.php إلى مجلد lib/ في مشروعك
3
هيّئ الـ Client
JavaScript
const { PayNextClient } = require('paynext-sdk');

const paynext = new PayNextClient({
  apiKey:    process.env.PAYNEXT_API_KEY,
  apiSecret: process.env.PAYNEXT_API_SECRET,
  baseUrl:   'https://paynext.one',
});
lib/paynext.js
// للخادم فقط — لا تستورده في مكوّنات المتصفح
import { PayNextClient } from 'paynext-sdk';

export const paynext = new PayNextClient({
  apiKey:    process.env.PAYNEXT_API_KEY,
  apiSecret: process.env.PAYNEXT_API_SECRET,
});
PHP
require_once 'vendor/autoload.php';
use PayNext\PayNextClient;

$paynext = new PayNextClient(
    apiKey:    $_ENV['PAYNEXT_API_KEY'],
    apiSecret: $_ENV['PAYNEXT_API_SECRET']
);
4
أنشئ جلسة دفع وأعِد التوجيه
عندما ينقر المستخدم على "اشترك"، استدعِ هذا على خادمك وأعِد توجيهه إلى الرابط المُعاد.
JavaScript
const session = await paynext.checkout.createSession({
  customer_id:   'CUS-XXXXXX',
  plan_id:       'PLAN-XXXXXX',
  billing_cycle: 'monthly',  // 'monthly' | 'yearly'
  success_url:   'https://yourapp.com/success',
  cancel_url:    'https://yourapp.com/pricing',
});

res.redirect(session.data.checkout_url);
5
استمع للـ Webhook لتفعيل المزايا
بعد نجاح الدفع، يُرسل PayNext حدث subscription.activated إلى نقطة نهايتك. استخدمه لفتح المزايا المميزة للمستخدم.
JavaScript

    req.headers['x-paynext-signature'],
    process.env.PAYNEXT_WEBHOOK_SECRET

  if (!valid) return res.sendStatus(401);

  const event = JSON.parse(req.body);
  if (event.type === 'subscription.activated') {
    // فعّل المزايا لـ event.data.customer.external_id
  }
  res.sendStatus(200);
});

المصادقة

يجب توقيع كل طلب API باستخدام API Key و Secret بخوارزمية HMAC-SHA256. يتولى SDK هذا تلقائيًا.

⚠️
API Secret للخادم فقط. لا تُعرّض PAYNEXT_API_SECRET للمتصفح أو التطبيق أو المستودع العام أبدًا. يجب إجراء جميع استدعاءات PayNext من خادمك الخلفي.

الترويسات المطلوبة في كل طلب

الترويسةالقيمة
AuthorizationBearer YOUR_API_KEY
X-SignatureHMAC-SHA256 لـ: METHOD\nPATH\nTIMESTAMP\nBODY
X-TimestampUnix timestamp بالثواني — الطلبات الأقدم من 5 دقائق مرفوضة

العملاء

العميل هو المستخدم النهائي لمنتجك. مرّر دائمًا معرّف المستخدم الخاص بك كـ external_id — هذا يتيح لك البحث عن عملاء PayNext دون تخزين معرّفات إضافية في قاعدة بياناتك.

✅
أفضل ممارسة: نمط الحصول أو الإنشاء. قبل إنشاء جلسة دفع، جرّب getByExternalId(userId) أولًا. أنشئ عميلًا جديدًا فقط إذا لم يكن موجودًا.
JavaScript
async function getOrCreate(user) {
  try {
    const res = await paynext.customers.getByExternalId(user.id);
    return res.data;
  } catch {
    const res = await paynext.customers.create({
      name:        user.name,
      email:       user.email,
      phone:       user.phone,       // مثال: '+966501234567'
      country:     user.country,     // ISO حرفان: 'SA', 'AE', 'EG'
      currency:    user.currency,    // ISO 3 أحرف: 'SAR', 'AED', 'EGP'
      external_id: String(user.id),  // معرّف المستخدم الخاص بك
    });
    return res.data;
  }
}
PHP
function getOrCreate($paynext, $user): array {
    try {
        return $paynext->customers->getByExternalId((string) $user['id'])['data'];
    } catch (PayNextException $e) {
        return $paynext->customers->create([
            'name'        => $user['name'],
            'email'       => $user['email'],
            'country'     => $user['country'] ?? 'SA',
            'currency'    => $user['currency'] ?? 'SAR',
            'external_id' => (string) $user['id'],
        ])['data'];
    }
}

الدفع والمدفوعات

يستضيف PayNext صفحة الدفع. تُنشئ جلسة، وتُعيد توجيه المستخدم، ويتولى PayNext نموذج البطاقة ومصادقة 3DS ومعالجة الدفع.

معاملات جلسة الدفع

المعاملالنوعمطلوبالوصف

الاشتراكات

تحقق من اشتراكات العملاء وأدِرها برمجيًا.

JavaScript
// التحقق من الاشتراك النشط
const subs   = await paynext.subscriptions.list({ customer_id: customer.id });
const active = subs.data.find(s => s.status === 'active');

// إلغاء في نهاية الفترة
await paynext.subscriptions.cancel(subscriptionId, { cancel_at_period_end: true });

// إيقاف مؤقت / استئناف
await paynext.subscriptions.pause(subscriptionId);
await paynext.subscriptions.resume(subscriptionId);

$subs   = $paynext->subscriptions->list(['customer_id' => $customerId]);
$active = null;
foreach ($subs['data'] as $sub) {
    if ($sub['status'] === 'active') { $active = $sub; break; }
}
$paynext->subscriptions->cancel($subscriptionId, ['cancel_at_period_end' => true]);

الـ Webhooks

تُعلم الـ Webhooks خادمك فوريًا عند وقوع أحداث — تفعيل اشتراك، نجاح دفع، إلخ. هكذا تعرف متى تفتح أو تسحب المزايا.

1اذهب إلى PayNext Admin ← Webhooks ← إضافة نقطة نهاية وأدخل رابط خادمك، مثلًا: https://yourapp.com/webhooks/paynext
2اختر الأحداث التي تريد استقبالها، ثم انسخ Webhook Secret المُنشأ
3أضف السر إلى بيئتك كـ PAYNEXT_WEBHOOK_SECRET
4أنشئ مسارًا على خادمك وتحقق من التوقيع في كل طلب وارد
⚠️
مهم: استخدم express.raw() في Node.js أو file_get_contents('php://input') في PHP لقراءة الجسم الخام قبل أي تحليل JSON. التحقق من التوقيع يتطلب البايتات الخام الدقيقة.
JavaScript
app.post('/webhooks/paynext',
  express.raw({ type: 'application/json' }),
  (req, res) => {
    const valid = PayNextClient.verifyWebhook(
      req.body,
      req.headers['x-paynext-signature'],
      process.env.PAYNEXT_WEBHOOK_SECRET
    );
    if (!valid) return res.status(401).end();

    const event = JSON.parse(req.body);
    switch (event.type) {
      case 'subscription.activated':
        // فعّل المزايا لـ event.data.customer.external_id
        break;
      case 'subscription.cancelled':
      case 'subscription.expired':
        // اسحب المزايا
        break;
      case 'payment.succeeded':
        // أرسل إيصال الدفع
        break;
    }
    res.sendStatus(200);
  }
);
app/api/webhooks/paynext/route.js
import { NextResponse } from 'next/server';
import { PayNextClient } from 'paynext-sdk';

export async function POST(request) {
  const rawBody   = await request.text();
  const signature = request.headers.get('x-paynext-signature');
  const valid = PayNextClient.verifyWebhook(
    rawBody, signature, process.env.PAYNEXT_WEBHOOK_SECRET
  );
  if (!valid) return NextResponse.json({ error: 'Invalid' }, { status: 401 });

  const event = JSON.parse(rawBody);
  // عالج event.type ...
  return NextResponse.json({ received: true });
}
PHP
$rawBody   = file_get_contents('php://input');

}
$event = json_decode($rawBody, true);

    case 'subscription.activated':
        // فعّل المزايا لـ $event['data']['customer']['external_id']
        break;
}
http_response_code(200);
echo json_encode(['received' => true]);

أنواع الأحداث

جميع الأحداث التي يمكن لـ PayNext إرسالها إلى نقطة نهاية الـ Webhook الخاصة بك.

subscription.activated
تم الدفع. الاشتراك نشط الآن.
subscription.created
تم إنشاء سجل اشتراك جديد.
subscription.renewed
تم تجديد دورة الفوترة بنجاح.
subscription.cancelled
تم إلغاء الاشتراك من المسؤول أو المستخدم.
subscription.expired
انتهت الفترة دون تجديد.
subscription.paused
تم إيقاف الاشتراك مؤقتًا.
subscription.resumed
تم استئناف الاشتراك الموقوف.
payment.succeeded
تم استلام الدفعة بنجاح.
payment.failed
فشلت محاولة الدفع.
payment.refunded
تم استرداد الدفعة.
